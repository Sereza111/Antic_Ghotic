## Gothic Anti-Detect (concept prototype)

Этот проект — экспериментальный инструмент для работы с несколькими браузерными профилями и управлением их параметрами (user‑agent, язык, часовой пояс и т.п.) через JS‑ориентированный UI в готическом стиле.

**Важно:** проект предназначен для легальных сценариев — тестирование, QA, исследование fingerprint‑техник и повышение личной приватности. Он **не гарантирует полный антидетект** и не предназначен для обхода антифрода конкретных сервисов.

### Структура (первая версия)

- `dev-todo-gothic.html` — референс готического UI (шрифты, фон, карточки, вкладки).
- `src/` — исходный код:
  - `src/frontend/` — SPA‑интерфейс.
  - `src/backend/` — Node.js API (профили, сценарии, Docker‑менеджер).

### Профили

Профиль описывает «виртуальное устройство» и включает:

- базовые браузерные параметры (user‑agent, язык, часовой пояс, размер экрана и др.);
- набор модулей fingerprint (Canvas/WebGL/WebRTC/Fonts и др.) с режимами `original / noise / spoof`;
- автоматизационные настройки (список сценариев, лимиты скорости, рандомизация задержек).

Формальное описание профиля находится в `src/backend/profile/profile.types.ts`.

### Docker и автоматизация (план)

Дальнейшая реализация предполагает:

- базовый Docker‑образ с Chromium + Node.js + Playwright/Puppeteer;
- отдельный контейнер на профиль (собственный user data dir, переменные окружения);
- API для запуска сценариев внутри контейнера.

### Как запустить docker-demo

1. Перейди в папку `docker`:
   - `cd docker`
2. Подними контейнер:
   - `docker compose up --build`

Контейнер запустит `src/backend/profile/profile-runner.js`, прочитает `docker/config/demo-profile.json`,
откроет `TARGET_URL` и сделает screenshot в volume `./artifacts`.

Профиль считается “изолированным” за счёт отдельного persistent user-data каталога на `/profiles/<PROFILE_ID>`,
а не за счёт того, что весь проект работает в контейнере целиком.

### Portainer (server stack) — backend + MySQL

В корне репозитория есть `docker-compose.yml` для Portainer.

**Важно:** MySQL выполнит SQL из `db/init/*.sql` **только при первом старте на пустом volume**.
Если ты уже поднимал MySQL и хочешь заново прогнать импорт — удали volume `antic_mysql_data`.

Переменные окружения, которые нужно задать в Portainer Stack:

- `MYSQL_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
